# Repository layout

This document explains the shape of the `df12-build` tree and the
responsibilities of its major paths. It is aimed at new contributors who need
to orient themselves quickly. For the documentation index, see
[contents.md](contents.md); for build and test workflows, see the
[developers guide](developers-guide.md).

`df12-build` holds workflow assets and operator guidance. It does not own
target-project source code. The workflow assets run against a separate project
that follows the df12 house conventions.

## Directory tree

The following tree is illustrative rather than exhaustive: it shows the paths a
contributor must understand quickly and omits generated caches and incidental
files. Paths marked as generated are produced by the build and must not be
edited by hand.

```plaintext
.
├── AGENTS.md               # Agent operating instructions for this repository
├── Makefile                # Canonical commit gates and build targets
├── README.md               # Project overview and quick start
├── docs/                   # Documentation set (see contents.md)
│   └── execplans/          # Self-contained execution plans
├── src/                    # TypeScript sources for the ODW workflow
│   └── workflows/df12-build-odw/
├── workflows/              # Built workflow artefacts (generated)
├── tests/                  # Module, behavioural, and property tests
│   ├── modules/            # Per-module unit and feature tests
│   ├── fixtures/           # Mock agents and repository fixtures
│   └── support/            # Shared test helpers
├── verify/                 # Dafny model and its verification companion
├── scripts/                # Build, spelling, and operator helper scripts
├── skills/                 # Operator skill documents
└── .github/workflows/      # Continuous Integration (CI) workflow definitions
```

## Path responsibilities

The table below describes the purpose and notable conventions of each major
path.

| Path | Purpose and conventions |
| --- | --- |
| `docs/` | The documentation set. New documents use the canonical filenames from the [documentation style guide](documentation-style-guide.md). |
| `docs/execplans/` | Long-lived execution plans, named after their branch or topic. |
| `src/workflows/df12-build-odw/` | Authoritative TypeScript sources for the ODW workflow. Edit behaviour here, not in `workflows/`. |
| `workflows/` | Generated workflow artefacts compiled from `src/`. Regenerated by `make workflow-build`; never edited by hand. |
| `tests/` | The test suite. `tests/modules/` holds unit and behavioural tests, `tests/fixtures/` holds mock agents and repository fixtures, and `tests/support/` holds shared helpers. |
| `verify/` | The Dafny recovery-decision model and its TypeScript companion. `*.dfy.gen` is generated. |
| `scripts/` | Build, spelling-rollout, and operator helper scripts. |
| `skills/` | Operator skill documents, such as the workshop supervisor playbook. |
| `.github/workflows/` | CI workflow definitions. |
| `Makefile` | The canonical entry point for formatting, linting, type checking, testing, and building. Prefer these targets over running tools directly. |

_Table 1: Responsibilities of the major repository paths._

## Generated and constrained paths

Some paths hold generated output and must be regenerated rather than edited:

- `workflows/df12-build-odw.js` is compiled from `src/workflows/df12-build-odw/`
  by `make workflow-build`. The `workflow-freshness` gate fails when the
  committed artefact and the sources diverge.
- `verify/recovery-decision.model.dfy.gen` is a generated companion to the
  Dafny model.
- `typos.toml` is regenerated during the Markdown gate from the shared
  en-GB-oxendict dictionary; the committed copy remains usable without network
  access.

Keep this document synchronized with the repository. Update it when the tree
changes enough that a contributor could otherwise follow outdated guidance.
